# LeadCognition Features & Benefits

## Core Value Propositions

### 1. GitHub Signal Capture
- **Feature**: Tracks GitHub events — stars, forks, commits, issues, PRs — across millions of repositories in real time
- **Benefit**: Know exactly which developers are actively engaged with your category, not just who has the right job title
- **Conversion Angle**: "See which developers starred your competitor's repo yesterday — and reach them first."

### 2. Developer Identity Enrichment
- **Feature**: Matches GitHub profiles to real identity: name, LinkedIn, current employer, job title
- **Benefit**: Go from anonymous username to contactable person with verified company context
- **Conversion Angle**: "Turn a GitHub username into a named, employed developer ready for outreach."

### 3. Email Discovery (Work + Personal)
- **Feature**: Surfaces verified work email (1 credit) and personal email (3 credits) for matched developers
- **Benefit**: Reach developers on their preferred channel — many use personal email for tools
- **Conversion Angle**: "Two email addresses per developer — work for formal outreach, personal for async tools."

### 4. Mobile Phone Unlock
- **Feature**: Mobile phone number unlock (10 credits) for high-value targets
- **Benefit**: Direct line for strategic accounts or warm follow-ups
- **Conversion Angle**: "For your top 20 prospects, skip the inbox entirely."

### 5. Credit-Based Unlock System
- **Feature**: Granular pay-per-unlock: identity (1cr), work email (1cr), personal email (3cr), mobile (10cr)
- **Benefit**: Only pay for what you need — no wasted spend on contacts you'll never use
- **Conversion Angle**: "500 credits might cover your entire month of meaningful outreach — not a firehose of junk data."

### 6. Domain-Based Company Discovery
- **Feature**: Drop in a company domain → get all developers associated with that company on GitHub
- **Benefit**: Instantly build a contact list for a target account or competitor's customer base
- **Conversion Angle**: "Type in any domain. Get every developer at that company in seconds."

### 7. Weekly Digest (Email + Slack)
- **Feature**: Automated weekly summary of new developer signals delivered to email or Slack
- **Benefit**: Stay on top of fresh intent without logging in every day
- **Conversion Angle**: "Your outbound queue builds itself. Get fresh developer signals in your inbox every Monday."

### 8. AI-Personalized Outreach
- **Feature**: Generates first-touch outreach copy based on the specific GitHub signal that triggered the contact
- **Benefit**: First message references what the developer actually did — dramatically higher reply rates
- **Conversion Angle**: "Your cold email mentions their actual commit. Not 'I noticed you work in infrastructure.'"

## Technical Features

### Data Pipeline
- **GitHub event ingestion**: Processes GitHub Archive data (millions of events/day)
- **Signal classification**: Filters for commercial-intent signals vs. general activity
- **Entity resolution**: Matches GitHub username → LinkedIn profile → verified email
- **Real-time enrichment**: New signals available within 24-48 hours

### Contact Enrichment
- **LinkedIn matching**: Profile URL, current title, current company, location
- **Email verification**: Verified deliverable emails, not just guesses
- **Phone lookup**: Mobile numbers for premium unlocks
- **Confidence scores**: Quality indicator on every enriched data point

### Delivery & Integration
- **Dashboard**: Web app at app.leadcognition.io — filter, sort, unlock, export
- **Weekly digest**: Email + Slack delivery
- **CSV export**: Direct import into your sequencer (Apollo, Instantly, Lemlist, etc.)
- **API**: REST API for automation (Scale plan)

## Competitive Differentiators

### vs. Apollo / ZoomInfo
- **GitHub signal layer**: Apollo has no behavioral intent data for developers; we do
- **Developer-specific**: Apollo's 200M contacts are mostly non-technical; ours are verified devs
- **Signal freshness**: Apollo contact data is static; our signals update daily

### vs. Hunter.io / RocketReach
- **Intent data**: Hunter finds emails; we find emails + show you WHY to contact someone now
- **Profile depth**: Hunter returns an email; we return LinkedIn, email, phone, and the triggering event
- **Discovery**: Hunter requires a name; we discover contacts from signals you didn't know about

### vs. Clay
- **Setup time**: Clay requires 10-20 hours of waterfall setup; LeadCognition is usable in 5 minutes
- **Cost**: Clay + its enrichment providers adds up fast; LeadCognition is all-in-one
- **GitHub-native**: Clay has no GitHub event layer; you'd have to build it yourself

### vs. Phantombuster
- **No-code**: Phantombuster requires agent config and scraping setup; we have a UI
- **Data quality**: Phantombuster scrapes LinkedIn; we match verified data sources
- **Enrichment**: Phantombuster scrapes profiles; we enrich with emails and phones
