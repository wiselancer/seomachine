# LeadCognition Style Guide

## Grammar & Formatting

### Capitalization
- **LeadCognition** — always CamelCase, never "Leadcognition" or "lead cognition"
- **GitHub** — capital G and H, never "Github"
- **B2B** — always capitalized
- **SDR/BDR/GTM** — always all-caps
- Product names: Apollo.io, Clay, Hunter.io, RocketReach (match official branding)

### Numbers
- Spell out one through nine; use digits for 10+
- Always use digits for: percentages (3%), prices ($49), credits (500 credits), measurements (30 days)
- Use commas for thousands: 2,000 credits, 200,000 developers

### Punctuation
- **Em dashes** without spaces: "signals — not contact data" → wrong: "signals—not contact data" → correct
- **Oxford comma**: always ("research, enrich, and reach")
- **Periods outside quotes**: American style unless quoting a full sentence

### Headlines
- Title case for H1 and H2
- Sentence case for H3 and below
- Include year in time-sensitive posts: "Best Developer Outreach Tools 2026"

## Content Formatting

### Code References
- Wrap API endpoints, field names, and code in backticks: `repo.stars_count`
- Use code blocks for any multi-line examples

### Lists
- Use bullet lists for unordered items (3+)
- Use numbered lists only for sequential steps
- Keep list items parallel grammatically
- Don't end list items with periods (unless they're full sentences)

### CTAs
- **Primary**: "Start free" or "Try LeadCognition free"
- **Secondary**: "See pricing", "View an example"
- Never: "Click here", "Learn more", "Check it out"
- CTA in square brackets with arrow: "[Start free →]"

## Blog Post Structure

### Standard Article
1. **Direct answer sentence** (1-2 sentences — answer the query before anything else)
2. **Hook** (the problem/why it matters — 2-3 sentences)
3. **Key Takeaways block** (3-5 bullets)
4. **Body** (H2 sections, each focused on one idea)
5. **Conclusion** (recap + CTA)

### Minimum Requirements
- 1,500 words minimum for blog posts
- 2,000–2,500 target for pillar content
- FAQ section (4–6 questions) at the end
- At least one link to app.leadcognition.io
- Author attribution: "By Semen Petrenko" or "By the LeadCognition Team"

## Voice Consistency Checklist

Before any piece goes live:
- [ ] No empty superlatives (powerful, robust, revolutionary, game-changing)
- [ ] Specific numbers used (not "many", "significantly", "dramatically")
- [ ] Technical terms used correctly (repo not "repository page", star not "favorite")
- [ ] CTA is specific and action-oriented
- [ ] Reads like a smart colleague, not a press release
